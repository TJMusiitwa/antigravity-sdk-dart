# Google Antigravity SDK Dart — Getting Started Examples 🚀

This directory contains getting-started examples for the **Google Antigravity Dart SDK**. These examples mirror the official Python SDK examples, demonstrating how to build powerful, stateful, and secure AI agents.

For advanced examples, see the [`deep_dives/`](deep_dives/README.md) directory.

---

## ⚙️ Prerequisites & Setup

Before running any of the examples, ensure you have set up your environment:

1. **Install Dart**: Make sure you have the [Dart SDK](https://dart.dev/get-dart) installed (version `>=3.11.5`).
2. **Set Gemini API Key**:
   ```bash
   export GEMINI_API_KEY="your_actual_gemini_api_key"
   ```

> [!NOTE]
> **Zero Configuration:** The SDK automatically detects your host OS and architecture on the first run, downloads the corresponding precompiled official `localharness` binary from PyPI, and extracts it using native tools (`unzip` on macOS/Linux, `tar` on Windows) under `~/.antigravity/bin/`. No manual download or CLI environment setup is required!

---

## 🏃 Running the Examples

From the root of the project (`antigravity`), run any specific example:

```bash
dart run example/getting_started/hello_world.dart
```

## 📚 Example Catalog

### 1. Hello World (`hello_world.dart`)
* **Focus**: The simplest agent flow.
* **Details**: Demonstrates creating a `LocalAgentConfig`, starting/stopping an `Agent`, and sending a simple chat prompt to await the complete response text.
* **Run Command**:
  ```bash
  dart run example/getting_started/hello_world.dart
  ```

### 2. Streaming (`streaming.dart`)
* **Focus**: Real-time response streaming.
* **Details**: Shows how to stream conversational response text tokens as they are generated by the model. It also illustrates streaming raw model thoughts/reasoning deltas and strongly-typed `ToolCall` events.
* **Run Command**:
  ```bash
  dart run example/getting_started/streaming.dart
  ```

### 3. Custom Tools (`custom_tools.dart`)
* **Focus**: Functional tool registration.
* **Details**: Illustrates how to register both stateless global functions and class-based stateful tools with the agent, which can then be dynamically called during agent execution.
* **Run Command**:
  ```bash
  dart run example/getting_started/custom_tools.dart
  ```

### 4. Hooks (`hooks.dart`)
* **Focus**: Intercepting the agent's lifecycle.
* **Details**: Demonstrates all 9 lifecycle hooks (e.g., `onSessionStart`, `onTurnStart`, `onToolCall`, `onToolResult`, `onTurnEnd`, etc.). Learn how to tap into raw telemetry, inject prompt context, or inspect intermediate state.
* **Run Command**:
  ```bash
  dart run example/getting_started/hooks.dart
  ```

### 5. Policies (`policies.dart`)
* **Focus**: Declarative tool & command safety.
* **Details**: Teaches the policy enforcement engine using priority-based declarative safety policies (`allow`, `deny`, and `askUser`) to restrict file reads, command execution, and MCP tool access.
* **Run Command**:
  ```bash
  dart run example/getting_started/policies.dart
  ```

### 6. Persistence (`persistence.dart`)
* **Focus**: State retention and session resumption.
* **Details**: Demonstrates using a disk-backed `saveDir` in the agent configuration to retain conversation history and step progress across agent restarts.
* **Run Command**:
  ```bash
  dart run example/getting_started/persistence.dart
  ```

### 7. Triggers (`triggers.dart`)
* **Focus**: Background event triggers.
* **Details**: Demonstrates running background workers and periodic tasks (using a custom `Duration` or standard cron syntax) that run alongside the agentic loop and inject system messages.
* **Run Command**:
  ```bash
  dart run example/getting_started/triggers.dart
  ```

### 8. Structured Output (`structured_output.dart`)
* **Focus**: Strongly-typed JSON schema compliance.
* **Details**: Teaches how to enforce structured response schemas (e.g. JSON matching a strict data model layout) using schema configuration properties, ensuring deterministic responses.
* **Run Command**:
  ```bash
  dart run example/getting_started/structured_output.dart
  ```

### 9. Observability (`observability.dart`)
* **Focus**: Metrics and token logging.
* **Details**: Shows how to register an audit hook to compute exact input/output tokens used per turn, monitor execution speeds, and inspect operational metrics.
* **Run Command**:
  ```bash
  dart run example/getting_started/observability.dart
  ```

### 10. Autonomous Shell (`autonomous_shell.dart`)
* **Focus**: High-privilege agentic terminal access.
* **Details**: Sets up a highly autonomous agent with fully approved shell and workspace permissions, demonstrating how an agent can perform multi-step engineering tasks under standard user monitoring.
* **Run Command**:
  ```bash
  dart run example/getting_started/autonomous_shell.dart
  ```

### 11. Multimodal (`multimodal.dart`)
* **Focus**: Ingestion of rich media files.
* **Details**: Demonstrates sending a mixed list of prompt items including plain text instructions, images (concrete `Image` subclass), and PDFs/documents (concrete `Document` subclass).
* **Run Command**:
  ```bash
  dart run example/getting_started/multimodal.dart
  ```

### 12. Human in the Loop (`human_in_the_loop.dart`)
* **Focus**: Stdin-based user confirmation hooks.
* **Details**: Integrates an interactive hook callback that captures the model's `AskQuestionEntry` options and prompts the human user on standard input (`stdin`) to confirm/select choices.
* **Run Command**:
  ```bash
  dart run example/getting_started/human_in_the_loop.dart
  ```

### 13. Agent Skills (`agent_skills.dart`)
* **Focus**: Loading and querying local agent skills.
* **Details**: Demonstrates how to point the agent to a folder containing local skills and query the agent about its active skill definitions.
* **Run Command**:
  ```bash
  dart run example/getting_started/agent_skills.dart
  ```

### 14. App Data Directory Override (`app_data_dir_override.dart`)
* **Focus**: Overriding the default storage location.
* **Details**: Shows how to configure `appDataDir` in `LocalAgentConfig` to use a custom directory path for intermediate scratch, logs, and downloaded harness resources.
* **Run Command**:
  ```bash
  dart run example/getting_started/app_data_dir_override.dart
  ```

### 15. Error Handling (`error_handler.dart`)
* **Focus**: Catching and resolving tool errors with hooks.
* **Details**: Illustrates registering an `OnToolErrorHook` subclass to catch exceptions thrown by tools and optionally feed a recovery response back to the agent to steer it.
* **Run Command**:
  ```bash
  dart run example/getting_started/error_handler.dart
  ```

### 16. Persona Config (`persona_config.dart`)
* **Focus**: Customizing system instructions and personas.
* **Details**: Demonstrates configuring templated persona identity additions alongside standard sections as well as fully overriding instructions via `CustomSystemInstructions`.
* **Run Command**:
  ```bash
  dart run example/getting_started/persona_config.dart
  ```

### 17. Subagents (`subagents.dart`)
* **Focus**: Spawning child subagents for task delegation.
* **Details**: Shows how to configure and spawn subagents within the agent configuration, showing how they execute tasks and logs output for tracking progression.
* **Run Command**:
  ```bash
  dart run example/getting_started/subagents.dart
  ```

### 18. MCP Tools (`mcp_tools.dart`)
* **Focus**: Connecting to external Model Context Protocol servers.
* **Details**: Demonstrates how to connect an agent to an external MCP server using the stdio transport. The pirate math MCP server (`example/resources/mcp_server.dart`) is spawned automatically as a child process via `McpStdioServer`. The agent discovers its tools at startup and can call them during the conversation.
* **Run Command**:
  ```bash
  dart run example/getting_started/mcp_tools.dart
  ```

### 19. Web Tools (`web_tools.dart`)
* **Focus**: Grounded real-time web search.
* **Details**: Demonstrates configuring an agent with the built-in `BuiltinTools.searchWeb` capability, enabling it to perform real-time Google Search queries and return grounded, sourced answers.
* **Run Command**:
  ```bash
  dart run example/getting_started/web_tools.dart
  ```
